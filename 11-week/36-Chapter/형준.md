# 36장 - 디스트럭처링 할당

디스트럭처링 할당은 구조화된 배열과 같은 이터러블 또는 객체를 `destructuring`하여 1개 이상의 변수에 개별적으로 할당하는 것을 말한다. 배열과 같은 이터러블 또는 객체 리터럴에서 필요한 값만 추출하여 변수에 할당할 때 유용하다.

## 배열 디스트럭처링 할당

ES5에서는 배열의 요소를 변수에 할당하기 위해 인덱스를 사용해야 했다.

```javascript
// ES5
var arr = [1, 2, 3];
var one = arr[0];
var two = arr[1];
var three = arr[2];
```

ES6의 배열 디스트럭처링 할당은 배열의 각 요소를 배열로부터 추출하여 1개 이상의 변수에 할당한다. 이때 **배열 디스트럭처링 할당의 대상은 이터러블이어야 하며, 할당 기준은 배열의 인덱스**다. 순서대로 할당한다.

```javascript
const arr = [1, 2, 3];
const [one, two, three] = arr;
console.log(one, two, three); // 1 2 3
```

배열 디스트럭처링 할당의 기준은 배열의 인덱스다. 이때 변수의 개수와 이터러블의 요소 개수가 반드시 일치할 필요는 없다.

```javascript
const [a, b] = [1, 2]; // 1 2
const [c, d] = [1]; // 1 undefined
const [e, f] = [1, 2, 3]; // 1 2
const [g, , h] = [1, 2, 3]; // 1 3 (요소 건너뛰기)
```

### 기본값 설정

배열 디스트럭처링 할당을 위한 변수에 기본값을 설정할 수 있다.

```javascript
const [a, b, c = 3] = [1, 2]; // 1 2 3
const [d, e = 5, f = 6] = [1]; // 1 5 6
```

### Rest 요소

배열 디스트럭처링 할당을 위한 변수에 Rest 파라미터와 유사하게 **Rest 요소** `...`을 사용할 수 있다. Rest 요소는 Rest 파라미터와 마찬가지로 반드시 마지막에 위치해야 한다.

```javascript
const [x, ...y] = [1, 2, 3];
console.log(x, y); // 1 [2, 3]
```

### 변수 교환과 함수 반환값

배열 디스트럭처링 할당을 사용하면 두 변수의 값을 쉽게 교환할 수 있고, 함수가 배열을 반환하는 경우 간편하게 값을 받을 수 있다.

```javascript
// 변수 교환
let a = 1,
  b = 2;
[a, b] = [b, a]; // 2 1

// 함수 반환값
function getValues() {
  return [1, 2, 3];
}
const [x, y, z] = getValues(); // 1 2 3
```

## 객체 디스트럭처링 할당

ES5에서 객체의 각 프로퍼티를 객체로부터 디스트럭처링하여 변수에 할당하기 위해서는 프로퍼티 키를 사용해야 했다.

```javascript
// ES5
var user = { firstName: "Ungmo", lastName: "Lee" };
var firstName = user.firstName;
var lastName = user.lastName;
```

ES6의 객체 디스트럭처링 할당은 객체의 각 프로퍼티를 객체로부터 추출하여 1개 이상의 변수에 할당한다. 이때 객체 디스트럭처링 할당의 대상은 객체이어야 하며, **할당 기준은 프로퍼티 키다.** 순서는 의미가 없으며 선언된 변수 이름과 프로퍼티 키가 일치하면 할당된다.

```javascript
const user = { firstName: "Ungmo", lastName: "Lee" };
const { lastName, firstName } = user;
console.log(firstName, lastName); // Ungmo Lee
```

### 변수 이름 변경과 기본값 설정

객체 디스트럭처링 할당을 할 때 변수 이름을 프로퍼티 키와 다르게 설정하거나 기본값을 설정할 수 있다.

```javascript
const user = { firstName: "Ungmo", lastName: "Lee" };
const { lastName: ln, firstName: fn } = user; // Ungmo Lee

// 기본값 설정
const { firstName = "Ungmo", lastName } = { lastName: "Lee" }; // Ungmo Lee

// 변수 이름 변경과 기본값 설정 함께 사용
const { firstName: fn = "Ungmo", lastName: ln } = { lastName: "Lee" }; // Ungmo Lee
```

### 중첩 객체 디스트럭처링

중첩된 객체도 디스트럭처링 할당이 가능하다.

```javascript
const user = {
  name: "Lee",
  address: { zipCode: "03068", city: "Seoul" },
};

const {
  address: { city },
} = user; // Seoul
const {
  address: { zipCode, city: cityName },
} = user; // 03068 Seoul
```

### Rest 프로퍼티

객체 디스트럭처링 할당을 위한 변수에 Rest 파라미터나 Rest 요소와 유사하게 **Rest 프로퍼티** `...`을 사용할 수 있다. Rest 프로퍼티는 Rest 파라미터나 Rest 요소와 마찬가지로 반드시 마지막에 위치해야 한다.

```javascript
const { x, ...rest } = { x: 1, y: 2, z: 3 };
console.log(x, rest); // 1 { y: 2, z: 3 }
```

## 함수 매개변수에서의 활용

객체 디스트럭처링 할당은 객체를 인수로 전달받는 함수의 매개변수에도 사용할 수 있다.

```javascript
// 기존 방식
function printTodo(todo) {
  console.log(
    `할일 ${todo.content}은 ${todo.completed ? "완료" : "비완료"} 상태입니다.`
  );
}

// 디스트럭처링 할당 사용
function printTodo({ content, completed }) {
  console.log(`할일 ${content}은 ${completed ? "완료" : "비완료"} 상태입니다.`);
}

// 기본값과 함께 사용
function printTodo({ content, completed = false }) {
  console.log(`할일 ${content}은 ${completed ? "완료" : "비완료"} 상태입니다.`);
}

// 배열 디스트럭처링도 함수 매개변수에서 사용 가능
function sum([a, b, c]) {
  return a + b + c;
}
console.log(sum([1, 2, 3])); // 6
```

## 실용적인 활용 예시

```javascript
// 객체에서 필요한 프로퍼티만 추출
const user = { id: 1, name: "Lee", age: 30, email: "lee@example.com" };
const { name, email } = user; // Lee lee@example.com

// 중첩 구조에서 값 추출
const response = {
  status: 200,
  data: { user: { id: 1, name: "Lee", profile: { age: 30, city: "Seoul" } } },
};
const {
  data: {
    user: {
      name,
      profile: { age, city },
    },
  },
} = response;
console.log(name, age, city); // Lee 30 Seoul

// 옵션 객체 패턴
function createUser({ name, age = 0, email = "", isActive = true }) {
  return { name, age, email, isActive };
}
const user = createUser({ name: "Lee", age: 30 });
// { name: 'Lee', age: 30, email: '', isActive: true }
```
