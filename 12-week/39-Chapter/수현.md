# 39장 - DOM

> HTML 문서의 계층적 구조와 정보를 표현하며 이를 제어할 수 있는 API, 즉 프로퍼티와 메서드를 제공하는 트리 자료구조

## 노드

html 요소의 어트리뷰트는 어트리뷰트 노드로, 콘텐츠는 텍스트 노드로 변환됨

HTML 문서는 HTML 요소들의 집합으로 이뤄지고 HTML 요소는 중첩 관계를 가짐. 이 말은 HTML 요소의 콘텐츠 영역에는 텍스트뿐만 아니라 다른 HTML 요소도 포함할 수 있음

### 트리 자료 구조

노드의 계층 구조로 이뤄진 자료 구조

부모 노드와 자식 노드로 구성괸 노드 간의 계층적 구조를 표현하는 비선형 자료구조

트리의 자료 구조는 하나의 최상위 노드에서 시작함.

최상위 노드는 부모 노드가 없으며 루트 노드, 루트 노드는 0개 이상의 자식 노드를 갖고 자식 노드가 없는 노드를 리프 노드라고 함.

**노드 객체들로 구성된 자료구조를 DOM이라함. (=DOM트리)**

DOM은 노드 객체의 계층적인 구조로 구성됨. 노드 객체에는 종류가 있고 상속구조를 가짐. 노드 객체는 총 12개의 종료가 있음.

**가장 중요한 노드 타입 4가지**

- 문서 노드
  - DOM 트리의 루트 노드
  - document객체로 나타냄
  - DOM트리의 진입점
- 요소 노드
  - HTML 요소를 가리키는 객체 (div, p, span 등)
  - 문서의 구조를 나타냄 자식노드로 다른 요소 노드나 텍스트 노드를 가질 수 있음
- 어트리뷰트 노드
  - HTML 요소의 어트리뷰트를 가리키는 객체
  - 요소 노드의 자식 노드가 아니라 요소에 속한 데이터임
  - 어트리뷰트 노드에 접근하려면 먼저 요소 노드에 접근해야 함.
- 텍스트 노드
  - HTML 요소의 텍스트를 가리키는 객체
  - 문서의 정보를 표현
  - 리프 노드

### 노드 객체의 상속 구조

**input 요소 노드 객체의 특성**

| input 요소 노드 객체의 특성                     | 프로토타입을 제공하는 객체 |
| ----------------------------------------------- | -------------------------- |
| 객체                                            | Object                     |
| 이벤트를 발생시키는 객체                        | EventTarget                |
| 트리 자료구조의 노드 객체                       | Node                       |
| 브라우저가 렌더링할 수 있는 웹 문서의 요소      | Element                    |
| 웹 문서의 요소 중에서 HTML 요소를 표현하는 객체 | HTMLElement                |
| HTML요소 중에서 input 요소를 표현하는 객체      | HTMLInputElement           |

**DOM은 HTML 문서의 계층적 구조와 정보를 표현하는 것은 물론 노드 객체의 종류, 즉 노드 타입에 따라 필요한 기능을 프로퍼티와 메서드의 집합인 DOM API로 제공함. 이 DOM API를 통해 HTML의 구조나 내용 또는 스타일 등을 동적으로 조작할 수 있음.**

## 요소 노드 취득

HTML의 구조나 내용 또는 스타일 등을 동적으로 저작하려면 먼저 요소 노드를 취득해야 함. 텍스트 노드는 요소 노드의 자식 노드이고 어트리뷰트 노드는 요소 노드와 연결되어 있기 때문에 텍스트 노드나 어트리뷰트 노드를 조작하고자 할 때도 마찬가지임.

### 1. **id를 이용한 요소 노드 취득**

- **정의**: `id` 속성은 HTML 요소의 고유한 식별자를 지정함. `id` 값을 사용하면 해당 요소를 빠르게 찾을 수 있음.
- **메서드**: `document.getElementById()`
- **특징**: `getElementById()`는 주어진 `id`를 가진 첫 번째 요소를 반환함. `id`는 페이지 내에서 유일해야 하므로 항상 하나의 요소만 반환됨.
- **예시**:
  ```jsx
  const element = document.getElementById("myId");
  console.log(element); // id가 'myId'인 요소 반환
  ```

### 2. **태그 이름을 이용한 요소 노드 취득**

- **정의**: `tag name`을 이용하여 문서 내에서 특정 태그 이름을 가진 요소들을 모두 선택할 수 있음. 여러 개의 요소가 반환될 수 있음.
- **메서드**: `document.getElementsByTagName()`
- **특징**: `getElementsByTagName()`은 주어진 태그 이름을 가진 모든 요소를 HTMLCollection으로 반환함. HTMLCollection은 실시간 컬렉션으로, DOM이 변경될 때 자동으로 업데이트됨.
- **예시**:
  ```jsx
  const paragraphs = document.getElementsByTagName("p");
  console.log(paragraphs); // 모든 <p> 태그를 포함하는 HTMLCollection 반환
  ```

### 3. **class를 이용한 요소 노드 취득**

- **정의**: `class` 속성으로 지정된 클래스 이름을 통해 해당 클래스를 가진 요소들을 선택할 수 있음.
- **메서드**: `document.getElementsByClassName()`
- **특징**: `getElementsByClassName()`은 주어진 클래스를 가진 모든 요소를 HTMLCollection으로 반환함. 이 역시 실시간 컬렉션.
- **예시**:
  ```jsx
  const elements = document.getElementsByClassName("myClass");
  console.log(elements); // 'myClass'를 가진 모든 요소 반환
  ```

### 4. **css 선택자를 이용한 요소 노드 취득**

- **정의**: CSS 선택자를 사용하여 요소를 선택하는 방법으로, 보다 복잡한 선택이 가능함.
- css 선택자
  스타일을 적용하고자 하는 html 요소를 특정할 때 사용하는 문법
- **메서드**: `document.querySelector()` 및 `document.querySelectorAll()`
- **특징**:
  - `querySelector()`는 주어진 CSS 선택자에 맞는 **첫 번째** 요소를 반환함.
  - `querySelectorAll()`은 주어진 CSS 선택자에 맞는 **모든** 요소를 반환함. 반환값은 `NodeList`
- **예시**:
  ```jsx
  const firstElement = document.querySelector(".myClass"); // 첫 번째 .myClass 요소 반환
  const allElements = document.querySelectorAll("p.myClass"); // 모든 <p> 태그 중 .myClass 클래스를 가진 요소 반환
  ```

### 5. **특정 요소 노드를 취득할 수 있는지 확인**

- **정의**: 인수로 전달한 css 선택자를 통해 특정 요소 노드를 취득할 수 있는지 확인함.
- **메서드**: `Element.prototype.matches` 메서드
- **특징**: 이벤트 위임을 사용할 때 유용

### 6. **HTMLCollection과 NodeList**

- **HTMLCollection**: `getElementsByTagName()` 또는 `getElementsByClassName()`을 사용하여 반환된 **실시간** 컬렉션임. DOM이 변경되면 컬렉션의 내용도 자동으로 업데이트됨.
  - 특징: 배열처럼 다룰 수 있지만, 배열 메서드(`forEach()` 등)는 지원하지 않음.
  - 예시:
    ```jsx
    const paragraphs = document.getElementsByTagName("p");
    console.log(paragraphs.length); // <p> 요소의 개수
    ```
  - 주의: 부작용 발생시킬 가능성이 있음
- **NodeList**: `querySelectorAll()`과 같은 메서드를 사용하여 반환되는 객체. 실시간 컬렉션은 아니며, 배열처럼 다룰 수 있는 경우도 있지만, 일부 메서드가 지원되지 않을 수 있음.
  - 특징: `NodeList`는 `forEach()`와 같은 배열 메서드를 지원함.
  - 예시:
    ```jsx
    const nodes = document.querySelectorAll("p");
    nodes.forEach((node) => {
      console.log(node); // 모든 <p> 태그 출력
    });
    ```
  - 주의: childNodes 프로퍼티가 반환하는 NodeList 객체는 HTMLCollection 객체와 같이 실시간으로 노드 객체의 상태 변경을 반영하는 live 객체로 동작하므로 주의가 필요함.
- 결론: 예상과 다르게 동작할 때가 있어 다루기 까다롭고 실수하기 쉬움. 따라서 노드 객체의 상태 변경과 상관없이 안전하게 DOM컬렉션을 사용하려면 HTMLCollection이나 NodeList객체를 배열로 변환하여 사용하는 것을 권장함.

## 노드 탐색

parentNode, previousSibling, childNodes 프로퍼티는 Node.prototype이 제공

Element가 포함된 previousElementSibling, nextElementSibling과 children프로퍼티는 Element.prototype이 제공함.

노드 탐색 프로퍼티는 모두 접근자 프로퍼티임.

### 1. **공백 텍스트 노드**

- **정의**: HTML 문서 내에서 태그 사이에 존재하는 공백, 줄 바꿈, 탭 등의 공백 문자를 텍스트 노드로 취급함. 이러한 공백 텍스트 노드는 DOM에 포함되며, `textContent` 또는 `innerText`로 접근할 수 있음.
- **특징**: 공백 텍스트 노드는 **보이지 않는 텍스트**로, 텍스트 내용이 없다고 생각할 수 있지만, 실제로는 DOM 트리에 존재하는 노드. 이런 공백 노드는 보통 `<div>`, `<p>` 등의 요소 사이에 자동으로 생성됨.
- **예시**:
  ```html
  <div>
    <p>Text</p>
    <p>More Text</p>
  </div>
  ```
  위의 HTML에서는 `<div>` 요소와 `<p>` 요소 사이에 공백 텍스트 노드가 존재. 이를 자바스크립트로 확인할 수 있음.
  ```jsx
  const div = document.querySelector("div");
  console.log(div.childNodes); // 공백 텍스트 노드를 포함한 자식 노드 목록 출력
  ```

### 2. **자식 노드 탐색**

- **정의**: 부모 노드의 자식 노드들을 탐색하는 방법. 자식 노드를 찾기 위해서는 `childNodes`, `children`, `firstChild`, `lastChild`와 같은 프로퍼티를 사용할 수 있음.
- 프로퍼티:
  - `Node.prototype.childNodes`: 부모 노드의 모든 자식 노드를 포함한 NodeList 반환. **childNodes 프로퍼티가 반환한 NodeList에는 요소 노드뿐만 아니라 텍스트 노드도 포함되어 있을 수 있음**
  - `Element.prototype.children`: 부모 노드의 요소 노드만 포함한 HTMLCollection 반환. **children 프로퍼티가 반환한 HTMLCollection에는 텍스트 노드가 포함되지 않음.**
  - `Node.prototype.firstChild`: 부모 노드의 첫 번째 자식 노드 반환. 텍스트 노드거나 요소 노드임
  - `Node.prototype.lastChild`: 부모 노드의 마지막 자식 노드 반환. 텍스트 노드거나 요소 노드임
  - `Element.prototype.firstElementChild` : 첫번째 자식 노드를 반환. 요소 노드만 반환
  - `Element.prototype.lastElementChild` : 마지막 자식 노드를 반환. 요소 노드만 반환

### 3. **자식 노드 존재 확인**

- **정의**: 특정 부모 노드에 자식 노드가 존재하는지 확인하는 방법. 자식 노드가 있으면 `true`, 없으면 `false`를 반환.
- **메서드**: `hasChildNodes()`
- **특징**: `hasChildNodes()` 메서드는 부모 노드에 자식 노드가 하나라도 있으면 `true`를 반환하고, 없으면 `false`를 반환. 텍스트 노드를 포함하여 자식 노드의 존재를 확인함. 텍스트 노드가 아닌 요소 노드가 존재하는지 확인하려면 `children.length`, `childElementCount` 프로퍼티로 확인
- **예시**:
  ```jsx
  const parent = document.getElementById("parent");
  if (parent.hasChildNodes()) {
    console.log("This node has child nodes.");
  } else {
    console.log("This node does not have any child nodes.");
  }
  ```

### 4. **요소 노드의 텍스트 노드 탐색**

- **정의**: 요소 노드 안에 포함된 텍스트 노드를 탐색하는 방법. 요소 노드 내부에 텍스트가 포함되어 있을 때, 그 텍스트는 텍스트 노드로 취급.
- 프로퍼티:
  - `firstChild`: 첫 번째 자식 노드가 텍스트 노드인 경우 텍스트 노드를 반환.
- **예시**:
  ```jsx
  const element = document.querySelector("div");
  console.log(element.firstChild); // 텍스트 노드 출력 (있다면)
  console.log(element.lastChild); // 마지막 자식 텍스트 노드 출력
  ```

### 5. **부모 노드 탐색**

- **정의**: 특정 노드의 부모 노드를 탐색하는 방법. 자식 노드에서 부모 노드를 찾아 올라가는 방식으로, DOM 트리에서 부모 노드에 접근할 수 있음.
- **메서드**:`Nodeprototype.parentNode`
- **특징**: 텍스트 노드는 dom트리의 최종단 노드인 리프노드이므로 부모 노드가 텍스트 노드인 경우는 없음.
- **예시**:
  ```jsx
  const child = document.querySelector("p");
  console.log(child.parentNode); // <div> 요소 (p 태그의 부모 요소) 반환
  ```

### **6. 형제 노드 탐색 (Sibling Node Traversal)**

- **정의**: 같은 부모를 가진 노드를 탐색하는 방법.
- 프로퍼티:
  - `Node.prototype.previousSibling`: 이전 형제 노드 반환. 텍스트 노드일 수도 있음
  - `Node.prototype.nextSibling`: 다음 형제 노드 반환. 텍스트 노드일 수도 있음
  - `Element.prototype.previousElementSibling` : 자신의 이전 형제 요소 노드를 탐색하여 반환
  - `Element.prototype.nextElementSibling` : 자신의 다음 형제 요소 노드를 탐색하여 반환
- **특징**: `previousSibling`과 `nextSibling`은 **텍스트 노드**까지 포함해서 반환하므로, 공백 텍스트 노드가 있을 수 있음.
- **예시**:
  ```jsx
  const sibling = document.querySelector("p");
  console.log(sibling.previousSibling); // 이전 형제 노드
  console.log(sibling.nextSibling); // 다음 형제 노드
  ```

## **노드 정보 취득 (Node Information Retrieval)**

- **정의**: 노드의 정보를 취득하는 방법.
- 프로퍼티:
  - `Node.prototype.nodeType`: 노드의 타입을 상수로 반환. 요소 노드는 `1`, 텍스트 노드는 `3`, 속성 노드는 `2` 등.
  - `Node.prototype.nodeName`: 노드의 이름을 문자열로 반환. 요소 노드의 경우 태그 이름을 반환. 텍스트 노드는 “#text”, 문서 노드는 “#document” 반환
  - `nodeValue`: 텍스트 노드의 값을 반환. 요소 노드는 `null`.
- **예시**:
  ```jsx
  const element = document.querySelector("div");
  console.log(element.nodeName); // 'DIV'
  console.log(element.nodeType); // 1 (Element)
  console.log(element.parentNode); // 부모 노드 반환
  ```

## **요소 노드의 텍스트 조작 (Text Manipulation in Element Nodes)**

**nodeValues**

Node.prototype.nodeValues 프로퍼티는 setter, getter 모두 존재하는 접근자 프로퍼티임. 참조와 할당 모두 가능.

- 특징: 노드 객체의 값을 반환함. 요소 노드에서는 `null` 반환. 텍스트 노드의 경우 텍스트 내용을 반환하거나 수정.
- 요소 노드의 텍스트를 변경하기 위한 순서
  1. 텍스트를 변경할 노드를 취득한 다음, 취득한 요소 노드의 텍스트 노드를 탐색한다. 텍스트 노드는 요소 노드의 자식 노드이므로 firstChild 프로퍼티를 사용하여 탐색한다.
  2. 탐색한 텍스트 노드의 nodeValue프로퍼티를 사용하여 텍스트 노드의 값을 변경한다.

**textContent**

요소 노드의 텍스트와 모든 자손 노드의 텍스트를 모두 취득하거나 변경함.

비슷한 동작을 하는 프로퍼티: `innerText` 사용하지 않는게 좋음

- innerText 프로퍼티는 css에 순종적임. css에 의해 비표시로 지정된 요소 노드의 텍스트를 반환하지 않음
- css를 고려해야 하므로 textContent 프로퍼티보다 느림.

## **DOM 조작 (DOM Manipulation)**

- **정의**: 새로운 노드를 생성하여 dom에 추가하거나 기존 노드를 삭제 또는 교체하는 것, 변경이 있을 시 리플로우 또는 리페인트가 발생하므로 성능에 영향을 주기에 조심해야 함.

### 1. **innerHTML**

- **정의**: 요소 노드의 HTML 마크업을 취득하거나 변경함.
- **특징**: HTML 태그와 텍스트 모두 포함하여 반환
- **주의사항**: xss 공격에 취약함. 악성 스크립트가 포함될 수 있어 사용자 입력을 그대로 삽입하는 데 사용 시 보안 취약점이 발생할 수 있음.
- **예시**:
  ```jsx
  const div = document.querySelector("div");
  console.log(div.innerHTML); // HTML 콘텐츠 가져오기
  div.innerHTML = "<p>New content</p>"; // HTML 콘텐츠 수정
  ```

### 2. **insertAdjacentHTML 메서드**

- **정의**: 지정된 위치에 HTML 콘텐츠를 삽입.
- **특징**: `beforebegin`, `afterbegin`, `beforeend`, `afterend` 네 가지 위치 옵션으로 삽입할 수 있음.
- **예시**:
  ```jsx
  const div = document.querySelector("div");
  div.insertAdjacentHTML("beforeend", "<p>New paragraph</p>"); // 끝에 HTML 삽입
  ```

### 3. **노드 생성과 추가**

- **정의**: 새로운 노드를 생성하고 부모 요소에 추가하는 방법.
- **메서드**:
  - `document.createElement()`: 새로운 요소 노드를 생성.
- 특징: 노드만 생성한 것이기 때문에 아무런 자식노드도 없고 연결된 노드도 없음.

- **메서드:** document.prototype.createTextNode(text)
- 특징: 텍스트 노드를 생성하여 반환.

- **메서드:** appendChild(childnode)
- 특징: 매개변수에게 인수로 전달한 노드를 appendchild메서드를 호출한 노드의 마지막 자식 노드로 추가함

이 과정을 거쳐야 새롭게 생성한 요소 노드가 dom에 추가됨.

### 4. **복수 노드 생성과 추가**

- **정의**: 여러 개의 노드를 생성하고 한 번에 부모 요소에 추가하는 방법.
- **메서드**: **document.createDocumentFragment**
- 특징: 리플로우 리페인팅이 1번 발생함

### 5. **노드 삽입**

- **정의**: 특정 위치에 노드를 삽입하는 방법.
- `Node.prototype.appendChild` 마지막 노드로 추가
- `Node.prototype.insertBefore` 지정한 위치에 노드 삽입

### 6. **노드 이동**

- **정의**: 노드를 부모 노드 내에서 다른 위치로 이동하는 방법.
- **메서드**: `parentNode.appendChild()` 또는 `parentNode.insertBefore()`.
- **예시**:
  ```jsx
  const divToMove = document.querySelector(".move");
  const container = document.getElementById("container");
  container.appendChild(divToMove); // divToMove를 마지막 자식으로 이동
  ```

### 7. **노드 복사**

- **정의**: 기존 노드를 복사하여 새로운 위치에 추가하는 방법.
- **메서드**:
  - `node.cloneNode(true)` (자식 노드도 함께 복사): 깊은 복사 (자식 포함).
  - `node.cloneNode(false)` (자식 노드 없이 복사): 얕은 복사 (자식 제외).
- **예시**:
  ```jsx
  const divToCopy = document.querySelector("div");
  const clonedDiv = divToCopy.cloneNode(true); // 깊은 복사
  document.body.appendChild(clonedDiv); // 복사된 div 추가
  ```

### 8. **노드 교체**

- **정의**: 기존 노드를 다른 노드로 교체하는 방법.
- **메서드**: `parentNode.replaceChild(newNode, oldNode)`
- **예시**:
  ```jsx
  const oldDiv = document.querySelector(".old");
  const newDiv = document.createElement("div");
  newDiv.textContent = "Replaced content";
  oldDiv.parentNode.replaceChild(newDiv, oldDiv); // oldDiv를 newDiv로 교체
  ```

### 9. **노드 삭제**

- **정의**: 노드를 삭제하는 방법.
- **메서드**: `parentNode.removeChild()`
- **예시**:
  ```jsx
  const divToRemove = document.querySelector(".remove");
  divToRemove.parentNode.removeChild(divToRemove); // div 삭제
  ```

## 어트리뷰트

모든 어트리뷰트 노드의 참조는 유사 배열 객체이자 이터러블인 NameNodeMap객체에 담겨서 요소 노드의 attributes 프로퍼티에 저장됨.

요소 노드의 모든 어트리뷰트 노드는 요소 노드의 Element.prototype.attributes 프로퍼티로 취득할 수 있음.

attributes 프로퍼티는 getter만 존재하는 읽기 전용 접근자 프로퍼티이며, 요소 노드의 모든 어트리뷰트 노드의 참조가 담긴 NameNodeMap 객체를 반환함.

### HTML 어트리뷰트 조작

어트리뷰트의 역할은 HTML 요소의 초기 상태를 지정하는 것, html 어트리뷰트 값은 html 요소의 초기 상태를 의미하며 이는 변하지 않음.

요소 노드는 2개의 상태를 관리해야함. (초기 상태/ 최신 상태)

초기 → 어트리뷰트 / 최신 → dom

**어트리뷰트 노드**

html어트리뷰트로 지정한 html 요소의 초기 상태는 어트리뷰트 노드에서 관리함.

**DOM 프로퍼티**

사용자가 입력한 최신 상태는 HTML 어트리뷰트에 대응하는 요소 노드의 DOM 프로퍼티가 관리함. DOM프로퍼티는 사용자의 입력에 의한 상태 변화에 반응하여 언제나 최신 상태를 유지함.

**HTML 어트리뷰트와 DOM 프로퍼티의 대응 관계**

- id 어트리뷰트와 id 프로퍼티는 1:1 대응하며, 동일한 값으로 연동함.
- input 의 value 어트리뷰트는 value 프로퍼티와 1:1 대응한다. 하지만 value 어트리뷰트는 초기 상태를 value 프로퍼티는 최신 상태를 갖는다.
- class 어트리뷰트는 className, classList 프로퍼티와 대응한다.
- for 어트리뷰트는 htmlFor 프로퍼티와 1:1 대응한다.
- td 요소의 colspan 어트리뷰트는 대응하는 프로퍼티가 존재하지 않는다.
- textContent 프로퍼티는 대응하는 어트리뷰트가 존재하지 않는다.
- 어트리뷰트 이름은 대소문자를 구별하지 않지만 대응하는 프로퍼티 키는 카멜 케이스를 따른다.

**DOM 프로퍼티 값의 타입**

문자열이 아닐수 있음

### data어트리뷰트와 dataset프로퍼티

data어트리뷰트와 dataset프로퍼티를 사용하면 html 요소에 정의한 사용자 정의 어트리뷰트와 자바스크립트 간에 데이터를 교환할 수 있음.

## 스타일

### 인라인 스타일 조작

[HTMLElement.prototype.style](http://HTMLElement.prototype.style) 프로퍼티는 setter, getter 모두 존재하는 접근자 프로퍼티로서 요소 노드의 인라인 스타일을 취득하거나 추가 또는 변경함.

### 클래스 조작

class 어트리뷰트에 대응하는 dom프로퍼티는 class가 아니라 className과 classList

**className**

Element.prototype.className 프로퍼티는 setter, getter 모두 존재하는 접근자 프로퍼티로서 html 요소의 class 어트리뷰트 값을 취득하거나 변경함.

**classList**

Element.prototype.classList 프로퍼티는 class 어트리뷰트의 정보를 담은 DOMTokenList객체를 반환함.

### 요소에 적용되어 있는 css 스타일 참조

style 프로퍼티는 인라인 스타일만 반환함. html 요소에 적용되어 있는 모든 css 스타일을 참조해야 할 경우 getComputedStyle메서드를 참조
