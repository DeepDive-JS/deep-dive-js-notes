# 21장 빌트인 객체

## 자바스크립트 객체의 분류

자바스크립트 객체는 3가지의 객체로 분류된다.

1. 표준 빌트인 객체

   ECMAScript 사양에 정의된 객체를 말하며, 애플리케이션 전역의 공통 기능을 제공한다.

   자바스크립트 실행 환경과 관계 없이 사용할 수 있고, 전역 객체의 프로퍼티로서 제공된다.

2. 호스트 객체

   ECMAScript 사양에 정의되어 있지 않지만 자바스크립트 실행 환경에서 추가로 제공하는 객체이다.

   브라우저 환경에선 DOM, BOM, SVG, Web Storage등과 같은 클라이언트 사이드 Web API를 호스트 객체로 제공한다.

3. 사용자 정의 객체

   앞의 두 객체와 다르게 기본 제공되는 객체가 아닌 사용자가 직접 정의한 객체를 말한다.

## 표준 빌트인 객체

Object, String, Number, Boolaen등 40여 개의 표준 빌트인 객체를 제공한다.

Math, Reflect, JSON을 제외한 객체는 인스턴스를 생성할 수 있는 생성자 함수 객체이다.

이는 프로토타입 메서드와 정적 메서드를 제공하고 생성자 함수 객체가 아닌 표준 빌트인 객체는 정적 메서드만 제공한다.

그렇기에 빌트인 객체가 생성한 인스턴스의 프로토타입은 표준 빌트인 객체의 prototype 프로퍼티에 바인딩된 객체다.

## 원시값과 래퍼 객체

원시값이 있는데 String, Number, Boolean같은 표준 빌트인 생성자 함수가 존재하는 이유는 무엇일까?

```jsx
const str = "hello";

// 원시 타입인 문자열이 프로퍼티와 메서드를 갖고 있는 객체처럼 동작한다.
console.log(str.length); // 5
console.log(str.toUpperCase()); // HELLO
```

원시값을 객체처럼 사용하면 엔진은 암묵적으로 연관된 객체를 생성하여 프로퍼티에 접근하거나 메서드를 호출하고 다시 원시값으로 되돌린다.

이처럼 문자열, 숫자, 불리언 값에 대해 객체처럼 접근하면 생성되는 임시 객체를 래퍼 객체라고 한다.

## 전역 객체

코드가 실행되기 이전 단계에 어떤 객체보다 먼저 생성되는 특수한 객체이며, 최상위 객체다.

브라우저에서는 window, Node.js에서는 global이 전역 객체를 가리킨다.

전역 객체는 표준 빌트인 객체, 호스트 객체, 전역 변수/전역 함수를 프로퍼티로 갖는다.

의도적으로 생성할 수 없으며, 전역 객체의 프로퍼티를 참조할 땐 window(global)을 생략할 수 있다.

```jsx
window.parseInt("F", 16); // 15

parseInt("F", 16); // 15

window.parsInt === parseInt; // true
```

var와 달리 let, const로 선언한 변수는 보이지 않는 개념적인 블록 내에 존재하게 되며, 따라서 전역 변수더라도 전역 객체의 프로퍼티가 아니게 된다.

### 빌트인 전역 프로퍼티

Infinity, NaN, undefined는 전역 객체의 프로퍼티를 의미하며, 주로 전역에서 사용하는 값을 제공한다.

```jsx
console.log(3/0); // Infinity
console.log(typeof Infinity); // number

console.log(Number("xyz")); // NaN
console.log(typeof NaN); // number

const foo;
console.log(foo); // undefined
console.log(typeof undefined); // undefined
```

### 빌트인 전역 함수

eval, isNaN, parseFloat, encodeURL/decodeURI등은 전역 에서 호출할 수 있는 빌트인 함수로서 전역 개체의 메서드다.

```jsx
eval("1 + 2;"); // 3
eval("var x = 5;"); // undefined
eval("({ a: 1 })"); // {a: 1}
eval("(function() { return 1; })"); // 1

parseFloat("3.14"); // 3.14
parseFloat("40 years"); // 40
parseFloat("He was 40"); // NaN

encodeURIComponent(uriComp); // name=%3D%CE9%...
decodeURIComponent(enc); // abc&job=program...
encodeURI(uriComp); // name=%%EC%..
decodeURI(enc); // name=lee&job=program...
```

### 암묵적 전역

```jsx
var x = 10;

function foo() {
  // 선언하지 않은 식별자에 값을 할당
  y = 20; // window.y = 20;
}

foo();

// 선언하지 않은 식별자 y를 전역에서 참조할 수 있다.
console.log(x + y); // 30
```

선언하지 않은 y는 마치 선언된 전역 변수처럼 동작한다.

선언하지 않은 식별자에 값을 할당하면 전역 객체의 프로퍼티가 되기 때문이다.

자바스크립트 엔진은 y = 20을 window.y = 20으로 해석하여 전역 객체에 프로퍼티를 동적 생성한다.

이를 암묵적 전역이라 한다.
