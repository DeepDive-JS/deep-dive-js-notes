# 23장 실행 컨텍스트

자바스크립트의 동작 원리를 담고 있는 핵심 개념이다.

4가지 타입의 소스코드는 실행 컨텍스트를 생성한다. 4가지 타입으로 구분하는 이유는 소스코드의 타입에 따라 실행 컨텍스트를 생성하는 과정과 관리 내용이 다르기 때문이다.

1. 전역 코드

   전역 변수를 관리하기 위해 최상위 스코프인 전역 스코프를 생성해야 한다.

   var 전역 변수와 함수 선언문으로 정의된 전역 함수를 전역 객체의 프로퍼티와 메서드로 바인딩하고 참조하기 위해 전역 객체와 연결되야 한다.

   이를 위해 전역 코드가 평가되면 전역 실행 컨텍스트가 생성된다.

2. 함수 코드

   지역 스코프를 생성하고 지역 변수, 매개변수, arguments 객체를 관리해야 한다. 그리고 생성한 지역 스코프를 전역 스코프에서 시작하는 스코프 체인의 일원으로 연결해야 한다.

   이를 위해 함수 코드가 평가되면 함수 실행 컨텍스트가 생성된다.

3. eval 코드

   엄격 모드에서 독자적인 스코프를 생성한다. 이를 위해 eval 코드가 평가되면 eval 실행 컨텍스트가 생성된다.

4. 모듈 코드

   모듈 코드는 모듈별로 독립적인 모듈 스코프를 생성한다. 이를 위해 모듈 코드가 평가되면 모듈 실행 컨텍스트가 생성된다.

## 소스코드의 평가와 실행

코드의 실행을 위해 평가와 실행 과정으로 나누어 처리한다.

평가 과정에선 실행 컨텍스트를 생성하고 변수, 함수 등의 선언문만 먼저 실행하여 생성된 변수나 함수 식별자를 키로 실행 컨텍스트가 관리하는 스코프에 등록한다.

평가 과정이 끝나면 선언문을 제외한 소스코드가 순차적으로 실행되는 런타임이 시작된다.

이때 변수나 함수의 참조를 실행 컨텍스트가 관리하는 스코프에서 검색해서 취득한다.

그리고 변수 값의 변경 등 소스코드의 실행 결과는 다시 실행 컨텍스트가 관리하는 스코프에 등록된다.

## 실행 컨텍스트 실행 순서

```jsx
const x = 1;

function foo() {
  const y = 2;

  function bar() {
    const z = 3;
    console.log(x + y + z);
  }

  bar();
}

foo();
```

1. 전역 컨텍스트 실행

   1.1. 전역 코드 평가

   전역 변수/함수인 x, foo를 등록하고 undefined로 초기화한다.(호이스팅)

   1.2. 전역 코드 실행

   전역 변수 x에 1이라는 값이 할당되고, 전역 함수 foo가 실행된다.

2. foo 함수 컨텍스트 실행

   2.1. 함수 코드 평가

   foo함수 코드인 y, bar 등록 → undefined 초기화

   2.2. 함수 코드 실행

   y에 2 할당 → bar 실행

3. bar 함수 컨텍스트 실행

   3.1. 함수 코드 평가

   z 등록 → 초기화

   3.2. 함수 코드 실행

   z에 3할당, console.log() 실행

4. bar 컨텍스트 종료
5. foo 컨텍스트 종료
6. 전역 컨텍스트 종료

## 실행 컨텍스트 구성

```jsx
const GlobalExecution = {
  Lexical_Environment: {
    Environment_Record: {
      Object_Environment_Record: "var, 함수 선언문",
      Declarative_Environment_Record: "const, let 함수/변수, class",
      This_Binding: global,
    },
    Outer_Environment_Reference: null,
  },
};

const FunctionExecution = {
  Lexical_Environment: {
    Environment_Record: "메개 변수, arguments, 변수",
    This_Binding: "할당된 this",
    Outer_Environment_Reference: "상위 컨텍스트의 렉시컬 환경",
  },
};
```

변수/함수가 담기는 환경 레코드에는 두 종류가 있는데 Object Env.Rec에는 ES6 이전 방식인 var, 함수 선언문으로 생성한 변수/함수들이 저장된다.

반대로 Declarative Env.Rec에는 ES6 문법인 const/let 함수 변수, class가 들어간다.

쉽게 말하자면 호이스팅 시 TDZ없이 undefined로 초기화되는 구식 문법이 Object Env.Rec에 저장되는 것이다.

## 실행 컨텍스트와 블록 레벨 스코프

```jsx
let x = 1;

if (true) {
  let x = 10;
  console.log(x);
}

console.log(x);
```

코드가 실행되면 전역 컨텍스트의 실행 단계에서 if문에 블록 시점에 진입하게 된다.

let, const는 블록 레벨 스코프이지만 함수처럼 컨텍스트를 새로 추가하는 것이 아닌 현재 실행중인 컨텍스트의 렉시컬 환경을 블록문이 끝나기 전까지 잠시 덮어씌운다.

그러면 if문 내부에선 전역 변수 x가 아닌, 블록 내부의 x를 사용해 console.log()를 실행하고 if문이 끝날 때 다시 원래의 전역 컨텍스트의 렉시컬 화면으로 복구되어 코드가 실행된다.
