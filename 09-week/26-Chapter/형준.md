# 26장 ES6 함수의 추가 기능

## 함수의 구분

ES6 이전의 함수는 일반 함수, 생성자 함수, 메서드로써 동일한 함수라도 다양한 형태로 호출할 수 있다.

즉, 사용 목적에 따라 명확이 구분되지 않는다.

모든 함수는 일반 함수로서 호출할 수 있는 것은 물론 생성자 함수로서 호출할 수 있다.

callable(호출 가능)이면서 constructor(생성 가능)다.

이는 모든 종류의 함수가 불필요할 수 있는 prototype 프로퍼티와 객체를 생성하는 것이다.

이런 문제를 해결하기 위해 ES6에서는 함수를 사용 목적에 따라 세 가지 종류로 구분했다.

| ES6 함수 구분 | constructor | prototype | super | argument |
| ------------- | ----------- | --------- | ----- | -------- |
| 일반 함수     | O           | O         | X     | O        |
| 메서드        | X           | X         | O     | O        |
| 화살표 함수   | X           | X         | X     | X        |

## 메서드

ES6 이전엔 메서드에 명확한 정의가 없었다.

일반적으로 객체에 바인딩된 함수를 일컫는 의미로 사용되었다.

ES6 사양에서 메서드는 메서드 축약 표현으로 정의된 함수만을 의미한다.

```jsx
const obj = {
  x: 1,
  // foo 메서드
  foo() {
    return this.x;
  },
  // bar 일반 함수
  bar: function () {
    return this.x;
  },
};
```

메서드는 인스턴스를 생성할 수 없고, 그렇기에 prototype를 생성하지 않는다.

그리고 자신을 바인딩한 객체를 가리키는 내부 슬롯 [[HomeObject]]를 갖는다.

super참조는 [[HomeObject]]를 사용하기에 메서드는 super 키워드를 사용할 수 있다.

반대로 메서드가 아닌 함수는 [[HomeObject]]를 갖지 않기 때문에 super 키워드를 사용할 수 없다.

## 화살표 함수

function 키워드 대신 화살표를 사용하여 기존보다 간략한 함수 정의가 된다.

표현만 간략한 것이 아닌, 내부 동작도 기존 함수보다 간략하다.

특히 콜백 함수 내부에서 this가 전역 객체를 가리키는 문제를 해결하기 위한 대안으로 유용하다.

```jsx
// 화살표 함수 기본 정의
const multiply = (x, y) => x * y;

// 매개변수 하나일 때 소괄호 생략
const arrow = x => { ... };

// 매개변수 없을 때 소괄호 생략 불가
const arrow = () => { ... };

// 함수 몸체가 하나의 문이라면 중괄호 생략
const power = x => x ** 2;

// 객체 리터럴 반환 시 객체 리터럴을 소괄호로 감싸야 함
const create = (id, content) => ({id, content});

// 즉시 실행 함수
const person = (name => ({
  sayHi() { return `Hi? My name is ${name}.`;}
}))('Lee');

// 고차 함수에 인수로 전달
[1, 2, 3].map(v => v * 2);
```

### 화살표 함수와 일반 함수의 차이

1. 화살표 함수는 인스턴스를 생성할 수 없음, prototype도 생성하지 않는다.
2. 중복된 매개변수 이름을 선언할 수 없다.
3. 화살표 함수는 함수 자체의 this, arguments, super, [new.target](http://new.target) 바인딩을 갖지 않는다.

   따라서 상위 스코프의 this, arguments, super, new.target을 참조

4. this

   콜백 함수로 일반 함수를 사용하고 this를 참조한다면 this는 undefined를 가리킨다.

   이는 일반 함수의 this는 무조건 undefined가 오기 때문이다.

   ```jsx
   Class Prefixer {
     constructor(prefix) {
       this.prefix = prefix
     }

     add(arr) {
       return arr.map(item => this.prefix + item);
     }
   }

   const prefixer = new Prefixer('-webkit-');
   console.log(prefixer.add(['transition', 'user-select']));
   // ['-webkit-transition', '-webkit-user-select']
   ```

   이때 화살표 함수는 함수 자체의 this 바인딩을 갖지 않는다.

   따라서 화살표 함수 내부에서 this를 참조하면 상위 스코프의 this를 그대로 참조한다.

   이를lexical this라 한다. 마치 렉시컬 스코프와 같이 화살표 함수의 this가 함수가 정의된 위치에 의해 결정된다는 것을 의미한다.

   만약 화살표 함수와 화살표 함수가 중첩되어 있다면 상위 화살표 함수에도 this 바인딩이 없으므로 스코프 체이닝을 거쳐 상위 함수 중, 화살표 함수가 아닌 함수의 this를 참조한다.

   최상위까지 체이닝이 이어질 경우 화살표 함수의 this는 전역 객체를 가리킨다.

5. super

   화살표 함수는 함수 자체의 super 바인딩을 갖지 않는다.

   this와 마찬가지로 상위 스코프의 super를 참조한다.

6. arguments

   마찬가지로 화살표 함수 자체의 arguments 바인딩을 갖지 않는다.

   상위 스코프의 arguments를 참조

## Rest 파라미터

Rest 파라미터는 매개변수 이름 앞에 세개의 점을 붙여 정의한 매개변수를 의미한다.

Rest 파라미터는 함수에 전달된 인수들의 목록을 배열로 전달받는다.

이름의 뜻대로 먼저 선언된 매개변수에 할당된 인수를 제외한 나머지 인수들로 구성된 배열이 할당된다.

따라서 Rest 파라미터는 반드시 마지막 파라미터이어야 하고 하나만 선언할 수 있다.

```jsx
function bar(p1, p2, ...rest) {
  console.log(p1); // 1
  console.log(p2); // 2
  console.log(rest); // [3, 4, 5]
}

bar(1, 2, 3, 4, 5);
```

## 매개변수 기본값

매개변수에 인수를 전달하지 않았을 때 대체되는 기본 값을 지정할 수 있다.

인수가 전달되지 않는 매개변수의 값은 undefined이기에 이는 예상치 못한 결과가 나올 수 있다.

즉, 확실하게 인수가 전달되지 않는 경우를 방지하는 방어 코드인 것

단 Rest 파라미터엔 기본 값을 지정할 수 없고

기본 값은 length, arguments 객체에 아무 영향을 주지 않는다.

```jsx
function logName(name = "Lee") {
  console.log(name);
}

logName(); // Lee
logName(undefined); // Lee
logName(null); // null
```
